### Root Cause Analysis

Проблема в структуре данных Google Sheets:
- В таблице Parking есть только P-004, P-006, P-008 со ссылками на партнёров
- Но сами партнёры (P-005, P-007, P-009) отсутствуют как отдельные строки в таблице
- Код искал партнёров в данных, не находил их, и рендерил первую парковку как regular

   - Оба тайла отображаются в tandem container
   - Строки 139-148: добавлен блок создания виртуального партнёра

2. **parking-properties.html** - аналогичная логика для деталей:
   - При парсинге parkingIds из URL создаются виртуальные объекты для отсутствующих парковок
   - Это позволяет отображать оба номера в заголовке "P-004 - P-005"

### Technical Solution

**Virtual Object Pattern** - создание временных объектов для missing data:
```javascript
if (!partner) {
    partner = {
        parking_id: parking.linked_id,
        type: 'tandem',
        linked_id: parking.parking_id
    };
}
```

Это решение позволяет работать с неполными данными в Google Sheets, где tandem парковки определены только для одной стороны пары.
